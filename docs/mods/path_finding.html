decitala.path_finding.path_finding_utils.best_source_and_sink(data)[source]¶

Calculates the “best” source and sink from a dataset based on two simple heuristics: (1) the fragment with the earliest (or latest, for sink) starting point, (2) the fragment with the greatest number of onsets.

Dijkstra path-finding algorithm from dynamic programming. Uses a min-heap data structure for efficiency.

  • weights (dict) – Dictionary with two keys and values (must sum to 1.0): "gap" and "onsets".

Calculates the distance and next matrices of the Floyd-Warshall path-finding algorithm.

Parameters

  • weights (dict) – weights to be used in the cost function. Must sum to 1. Requires “gap” and “onsets” values.

  • verbose (bool) – Whether to log messages (including showing a progress bar).

Returns

tuple

decitala.path_finding.pofp.check_break_point(data, i)[source]¶

Helper function for get_break_points. Checks index i of the onset_list that all values prior to it are less than or equal to \(b_i\) and \(s_i\). If True, this means that the data at index i is >= all previous.

Parameters

  • data (list) – data from rolling_search.

  • i (int) – index of the data to check.

Returns

whether or not the queried index is a break point.

Return type

bool

>>> data = [
...             {"fragment": "info1", "mod": ("r", 1.0), "onset_range": (0.0, 2.0)},
...     {"fragment": "info2", "mod": ("r", 2.0), "onset_range": (0.0, 4.0)},
...     {"fragment": "info3", "mod": ("d", 0.25), "onset_range": (2.0, 4.0)},
...     {"fragment": "info4", "mod": ("rd", 0.25), "onset_range": (2.0, 5.75)},
...     {"fragment": "info5", "mod": ("r", 3.0), "onset_range": (2.5, 4.5)},
...     {"fragment": "info6", "mod": ("r", 1.0), "onset_range": (4.0, 5.5)},
...     {"fragment": "info7", "mod": ("rd", 0.25), "onset_range": (6.0, 7.25)}
... ]
>>> print(check_break_point(data, 2))
False
>>> print(check_break_point(data, 6))
